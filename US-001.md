Bạn đang sửa project `coding-flow-v0.3`.

Mục tiêu: nâng cấp lên `coding-flow-v0.4`.

## Mục tiêu chính

Thêm cơ chế để `cflow` có thể gọi agent oneshot cho các bước nặng, nhằm giữ main context sạch.

Flow hiện tại:

```text
request -> plan -> verify -> ship
```

Flow mới:

```text
request -> agent plan -> agent coding -> verify -> ship
```

Ngoài ra vẫn giữ các command thủ công:

```text
request -> plan -> coding -> verify -> ship
```

Trong đó:

* `cflow plan` = nhận plan JSON transient từ stdin và render `PLAN.md`
* `cflow coding` = nhận coding JSON transient từ stdin và render `CODING.md`
* `cflow agent plan` = gọi agent oneshot để sinh plan JSON, sau đó render `PLAN.md`
* `cflow agent coding` = gọi agent oneshot để implement code, nhận coding JSON, sau đó render `CODING.md`

---

## Nguyên tắc bắt buộc

1. Không lưu JSON trong `.coding/`.
2. JSON chỉ là transient input/output qua stdin hoặc stdout.
3. Markdown là artifact duy nhất được giữ lại.
4. Mỗi task nằm trong folder riêng:

```text
.coding/tasks/<task-id>/
```

5. CLI `cflow` là bên duy nhất render markdown artifact.
6. Agent/LLM không được sửa trực tiếp các artifact:

```text
REQUEST.md
PLAN.md
CODING.md
VERIFY.md
SHIP.md
```

7. Main session chỉ nhận output ngắn từ CLI.
8. Không dump toàn bộ plan, diff, log dài, hoặc stdout thô của agent vào main context.
9. Không ship nếu verify chưa `passed`.
10. Không commit nếu user chưa yêu cầu rõ.

---

## Cấu trúc artifact mới

Mỗi task folder có dạng:

```text
.coding/
├── current
└── tasks/
    └── <task-id>/
        ├── REQUEST.md
        ├── PLAN.md
        ├── CODING.md
```

Không được tạo hoặc giữ:

# Command cần thêm

## 1. `cflow coding --task current`

Command này là core command thủ công, tương tự `request`, `plan`, `verify`, `ship`.

### Behavior

Đọc JSON từ stdin hoặc `--input`, validate, render:

```text
.coding/tasks/<task-id>/CODING.md
```

Không lưu JSON.

Nếu thiếu `--task`, mặc định dùng `current`.

### Input JSON schema

```json
{
  "status": "ready_for_verify",
  "summary": [],
  "changed_files": [],
  "notes": [],
  "next": "verify"
}
```

Allowed `status`:

```text
ready_for_verify
blocked
partial
failed
```

Allowed `next`:

```text
verify
plan
clarify
none
```

### Render template

Tạo file:

```text
templates/coding.md
```

Nội dung:

```md
# Coding

## Status

{{status}}

## Summary

{{summary}}

## Changed Files

{{changed_files}}

## Notes

{{notes}}

## Next

{{next}}
```

### Ví dụ dùng

```bash
cat <<'JSON' | node bin/cflow coding --task current
{
  "status": "ready_for_verify",
  "summary": [
    "Implemented focus session timer",
    "Added garden persistence",
    "Added clear garden action"
  ],
  "changed_files": [
    "src/App.tsx",
    "src/focusGarden.ts"
  ],
  "notes": [
    "No backend added",
    "localStorage key: focus_garden_sessions"
  ],
  "next": "verify"
}
JSON
```

## 2. `cflow agent plan --task current`

Command này gọi agent oneshot để lập plan.

### Behavior

1. Resolve task giống các command khác.
2. Require `REQUEST.md` tồn tại.
3. Đọc `REQUEST.md`.
4. Build prompt cho planning agent.
5. Gọi external agent command.
6. Agent phải trả về valid plan JSON only.
7. Parse stdout thành JSON.
8. Validate JSON giống `cflow plan`.
9. Render:

```text
.coding/tasks/<task-id>/PLAN.md
```

10. Không lưu plan JSON.
11. Stdout của `cflow agent plan` chỉ in summary ngắn.

### Output cho main context

Chỉ in dạng ngắn:

```text
Plan created: .coding/tasks/<task-id>/PLAN.md
Implementation steps: <number>
Files expected: <number>
Next: cflow agent coding --task current
```

Không in toàn bộ PLAN.md.

Không in stdout thô của agent.

---

## 3. `cflow agent coding --task current`

Command này gọi agent oneshot để implement code theo plan.

### Behavior

1. Resolve task giống các command khác.
2. Require `PLAN.md` tồn tại.
3. Đọc `PLAN.md`.
4. Build prompt cho coding agent.
5. Gọi external agent command.
6. Agent được phép sửa code project.
7. Agent không được sửa các artifact markdown trong `.coding/tasks/<task-id>/`.
8. Agent phải trả về valid coding JSON only.
9. Parse stdout thành JSON.
10. Validate coding JSON.
11. Render:

```text
.coding/tasks/<task-id>/CODING.md
```

12. Không lưu coding JSON.
13. Stdout của `cflow agent coding` chỉ in summary ngắn.

### Output cho main context

Chỉ in dạng ngắn:

```text
Coding completed: .coding/tasks/<task-id>/CODING.md
Status: ready_for_verify
Changed files: <number>
Next: cflow verify --task current
```

Không in diff dài.

Không in toàn bộ CODING.md.

Không chạy verify.

Không commit.

# Agent provider

Không cần implement provider phức tạp ở v0.4.

Thêm env:

```text
CFLOW_AGENT_CMD
```

Nếu `CFLOW_AGENT_CMD` không set, default dùng:

```text
codex exec
```

Thiết kế helper:

```js
runAgent(promptText, options)
```

Nhiệm vụ:

1. Gọi external command.
2. Truyền prompt vào stdin hoặc argument phù hợp.
3. Capture stdout.
4. Parse JSON từ stdout.
5. Nếu stdout không phải JSON hợp lệ, reject với error rõ.
6. Không ghi stdout thô vào artifact.
7. Không lưu JSON output vào `.coding/`.

Có thể để implementation đơn giản v0.4:

```text
CFLOW_AGENT_CMD="codex exec"
```

Hoặc cho phép override:

```bash
CFLOW_AGENT_CMD="codex exec --ask-for-approval never" node bin/cflow agent plan --task current
```

# Prompt cho agent plan

Tạo file skill:

```text
skills/agent-plan.md
```

Nội dung cần có:

```md
---
name: cflow:agent-plan
description: Generate a code plan JSON from REQUEST.md using a oneshot agent.
argument-hint: "[--task current]"
allowed-tools:
  - Read
  - Bash
requires:
  - cflow
---

# cflow:agent-plan

## Purpose

Create a 9-section code plan from the current task's REQUEST.md.

## Rules

- Read REQUEST.md.
- Do not edit code.
- Do not edit markdown artifacts.
- Do not create JSON files.
- Return valid JSON only.
- cflow will validate and render PLAN.md.

## Required plan sections

1. Objective
2. Scope
3. Requirements
4. Technical Approach
5. Files to Change
6. Implementation Steps
7. Test Plan
8. Risks
9. Done Criteria

## Required fields

The output JSON must meaningfully fill:

- objective
- scope
- requirements
- technical_approach
- done_criteria.items

## Output JSON

Return only JSON matching:

{
  "objective": "",
  "scope": {
    "in": [],
    "out": []
  },
  "requirements": [],
  "technical_approach": [],
  "files_to_change": [],
  "implementation_steps": [
    {
      "text": "",
      "status": "todo"
    }
  ],
  "test_plan": {
    "planned": [],
    "result": []
  },
  "risks": [],
  "done_criteria": {
    "items": [],
    "verified": []
  }
}
```

# Prompt cho agent coding

Tạo file skill:

```text
skills/agent-coding.md
```

Nội dung cần có:

```md
---
name: cflow:agent-coding
description: Implement code from PLAN.md using a oneshot agent and return a coding summary JSON.
argument-hint: "[--task current]"
allowed-tools:
  - Read
  - Edit
  - MultiEdit
  - Bash
requires:
  - cflow
---

# cflow:agent-coding

## Purpose

Implement the current task according to PLAN.md.

## Rules

- Read PLAN.md.
- Implement the code changes described in the plan.
- Do not edit REQUEST.md.
- Do not edit PLAN.md.
- Do not edit CODING.md.
- Do not edit VERIFY.md.
- Do not edit SHIP.md.
- Do not create JSON files.
- Do not run ship.
- Do not commit.
- Return valid JSON only.
- cflow will validate and render CODING.md.
- Verification is handled by the verify phase.

## Output JSON

Return only JSON matching:

{
  "status": "ready_for_verify",
  "summary": [],
  "changed_files": [],
  "notes": [],
  "next": "verify"
}

Allowed status:

- ready_for_verify
- blocked
- partial
- failed

Allowed next:

- verify
- plan
- clarify
- none
```


Tức là `cmd = agent`, `subcmd = plan|coding`.

```text
taskDir = resolveTask(taskArg)
```


# Update AGENTS.md

Rút gọn và cập nhật core instruction:

````md
# Agent Instructions

## Flow

Use this flow for coding work:

```text
request -> agent plan -> agent coding -> verify -> ship
````

Manual fallback:

```text
request -> plan -> coding -> verify -> ship
```

## Core Rules

* Always create/select a task first with `cflow new "<task-name>"`.
* Use `.coding/tasks/<task-id>/` for task artifacts.
* Do not create or keep `.coding/*.json` files.
* Use JSON only as transient stdin/stdout input for `cflow`.
* Do not edit artifact markdown directly.
* Let `cflow` render:

  * REQUEST.md
  * PLAN.md
  * CODING.md
  * VERIFY.md
  * SHIP.md
* Prefer `cflow agent plan --task current` for non-trivial planning.
* Prefer `cflow agent coding --task current` for non-trivial implementation.
* Keep main context clean: rely on short `cflow` summaries.
* Do not read long artifacts unless debugging or explicitly needed.
* Do not ship unless `VERIFY.md` exists and verification status is `passed`.
* Use `--dry-run` before `--commit`.
* Commit only when the user explicitly asks to commit.

