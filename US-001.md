Bạn đang sửa project `coding-flow-v0.2`.

Mục tiêu: nâng cấp lên `coding-flow-v0.3`.

## Mục tiêu chính

Hiện tại CLI đang dùng cấu trúc cố định:

```text
.coding/
├── request.json
├── plan.json
├── verify.json
├── ship.json
├── REQUEST.md
├── PLAN.md
├── VERIFY.md
└── SHIP.md
```

Thiết kế mới phải đổi sang:

```text
.coding/
├── current
└── tasks/
    └── <task-id>/
        ├── REQUEST.md
        ├── PLAN.md
        ├── VERIFY.md
        └── SHIP.md
```

Không được lưu giữ các file JSON trong `.coding/`.

JSON chỉ là input tạm thời qua stdin hoặc `--input`, sau khi render xong không được copy hoặc ghi JSON vào `.coding/`.

---

## Nguyên tắc bắt buộc

1. Markdown là artifact duy nhất được giữ lại.
2. JSON là transient input, không phải artifact.
3. CLI `cflow` là bên duy nhất render markdown.
4. Agent/LLM không được sửa trực tiếp các file markdown artifact.
5. Mỗi task phải có folder riêng để không ghi đè task cũ.
6. `cflow ship --commit` chỉ được commit khi verify đã `passed`.
7. Mặc định ship nên chạy `--dry-run`, commit thật chỉ khi có flag `--commit`.

## CLI cần hỗ trợ

### 1. `cflow new "<task-name>"`

Tạo task mới.

Ví dụ:

```bash
node bin/cflow new "focus garden"
```

Phải tạo folder:

```text
.coding/tasks/<task-id>/
```

Trong đó `<task-id>` có dạng:

```text
YYYYMMDD-HHMMSS-slug
```

Ví dụ:

```text
20260602-153012-focus-garden
```

Đồng thời ghi file:

```text
.coding/current
```

Nội dung của `.coding/current` là relative path đến task folder:

```text
tasks/20260602-153012-focus-garden
```

Sau khi chạy xong, in ra task id và path.

---

### 2. `cflow status`

Hiển thị current task.

Nếu chưa có `.coding/current`, báo rõ là chưa có current task và gợi ý chạy:

```bash
cflow new "<task-name>"
```

Nếu có current task, hiển thị:

```text
Current task: <task-id>
Path: .coding/tasks/<task-id>
Artifacts:
- REQUEST.md: exists/missing
- PLAN.md: exists/missing
- VERIFY.md: exists/missing
- SHIP.md: exists/missing
```

---

### 3. `cflow request --task current`

Đọc JSON từ stdin, validate, render:

```text
.coding/tasks/<task-id>/REQUEST.md
```

Không ghi `request.json`.

Vẫn cho phép fallback:

```bash
cflow request --task current --input /tmp/request.json
```

Nhưng không được copy file input vào `.coding/`.

Nếu thiếu `--task`, mặc định dùng `current`.

---

### 4. `cflow plan --task current`

Đọc JSON từ stdin hoặc `--input`, validate, render:

```text
.coding/tasks/<task-id>/PLAN.md
```

Không ghi `plan.json`.

Nếu thiếu `--task`, mặc định dùng `current`.

Validation bắt buộc cho initial plan:

```text
objective
scope
requirements
technical_approach
done_criteria.items
```

Nếu thiếu hoặc rỗng thì reject với error rõ ràng.

---

### 5. `cflow verify --task current`

Đọc JSON từ stdin hoặc `--input`, validate, render:

```text
.coding/tasks/<task-id>/VERIFY.md
```

Không ghi `verify.json`.

Nếu thiếu `--task`, mặc định dùng `current`.

Allowed `status`:

```text
passed
failed
partial
skipped
```

Nếu status không hợp lệ thì reject.

---

### 6. `cflow ship --task current --dry-run`

Đọc JSON từ stdin hoặc `--input`, validate, render:

```text
.coding/tasks/<task-id>/SHIP.md
```

Không ghi `ship.json`.

Nếu thiếu `--task`, mặc định dùng `current`.

Ship phải reject nếu:

```text
ready != true
verification.status != passed
VERIFY.md chưa tồn tại trong task folder
```

`--dry-run` phải chạy:

```bash
git status --short
```

Không commit.

---

### 7. `cflow ship --task current --commit`

Chỉ commit nếu:

```text
ready = true
verification.status = passed
VERIFY.md tồn tại
```

Sau đó:

```bash
git add .
git commit -m "<conventional commit message>"
```

Commit message lấy từ:

```json
{
  "commit": {
    "type": "feat",
    "scope": "focus-garden",
    "message": "add focus garden mvp"
  }
}
```

Render thành:

```text
feat(focus-garden): add focus garden mvp
```

Nếu `scope` rỗng:

```text
feat: add focus garden mvp
```

## JSON input behavior

Tất cả command phase sau phải đọc JSON theo thứ tự ưu tiên:

1. Nếu có `--input <file>`, đọc JSON từ file đó.
2. Nếu không có `--input`, đọc JSON từ stdin.
3. Nếu stdin rỗng hoặc JSON invalid, báo lỗi rõ.

Áp dụng cho:

```text
request
plan
verify
ship
```

Không áp dụng cho:

```text
new
status
```

Không bao giờ ghi JSON vào `.coding/`.

## Task resolution

Implement helper `resolveTask(taskArg)`.

Rules:

```text
--task current hoặc thiếu --task:
  đọc .coding/current

--task <task-id>:
  dùng .coding/tasks/<task-id>

--task .coding/tasks/<task-id>:
  dùng path đó trực tiếp
```

Nếu task không tồn tại, báo lỗi rõ.

Không render vào `.coding/REQUEST.md`, `.coding/PLAN.md`, `.coding/VERIFY.md`, `.coding/SHIP.md` nữa.

## README cần cập nhật

Cập nhật README theo flow mới:

```bash
node bin/cflow new "focus garden"

cat <<'JSON' | node bin/cflow request --task current
{ ... }
JSON

cat <<'JSON' | node bin/cflow plan --task current
{ ... }
JSON

cat <<'JSON' | node bin/cflow verify --task current
{ ... }
JSON

cat <<'JSON' | node bin/cflow ship --task current --dry-run
{ ... }
JSON
```

README cần nói rõ:

```text
- JSON chỉ là transient input.
- CLI không lưu JSON.
- Markdown trong .coding/tasks/<task-id>/ là artifact chính.
- Mỗi task có folder riêng nên không ghi đè task cũ.
```

## AGENTS.md cần rút gọn/cập nhật

Cập nhật `AGENTS.md` với các rule ngắn gọn:

```text
Flow: request -> plan -> implement -> verify -> ship

Always create/select a task first:
cflow new "<task-name>"

Do not create or keep .coding/*.json files.

Use JSON only as stdin input to cflow.

Persistent artifacts are only:
- .coding/tasks/<task-id>/REQUEST.md
- .coding/tasks/<task-id>/PLAN.md
- .coding/tasks/<task-id>/VERIFY.md
- .coding/tasks/<task-id>/SHIP.md

Do not edit these markdown artifacts directly.
Use cflow to render them.

Do not ship unless VERIFY.md exists and verification status is passed.

Use --dry-run before --commit.
Commit only when the user explicitly asks to commit.
```

## Skills cần cập nhật

Cập nhật các file trong `skills/`:

```text
skills/request.md
skills/plan.md
skills/verify.md
skills/ship.md
```

Điểm cần sửa:

1. Không hướng dẫn tạo `.coding/*.json`.
2. Hướng dẫn LLM sinh JSON rồi pipe trực tiếp vào CLI.
3. Command mẫu dùng:

```bash
cat <<'JSON' | cflow <phase> --task current
...
JSON
```

4. Nhấn mạnh JSON không được lưu.
5. Markdown artifact chỉ được tạo bởi CLI.

## Examples cần cập nhật

Hiện tại examples có thể đang chứa:

```text
examples/request.json
examples/plan.json
examples/verify.json
examples/ship.json
```

Có thể giữ lại examples nếu muốn, nhưng README phải nói đây chỉ là input mẫu.

Tốt hơn cho v0.3:

```text
examples/
├── request.stdin.sh
├── plan.stdin.sh
├── verify.stdin.sh
└── ship.stdin.sh
```

Mỗi file là một script mẫu dùng heredoc pipe vào CLI.

Ví dụ:

```bash
#!/usr/bin/env bash
cat <<'JSON' | node bin/cflow request --task current
{
  "summary": "Build Focus Garden MVP",
  "type": "new_feature",
  "planning_needed": true,
  "lane": "normal",
  "risk_flags": [],
  "hard_gates": [],
  "assumptions": [],
  "clarifying_questions": [],
  "next_action": "plan"
}
JSON
```

## Backward compatibility

Không cần giữ backward compatibility với output cũ.

Có thể vẫn giữ `--input` để tiện test, nhưng output path phải là task folder mới.

Không được còn hành vi ghi vào:

```text
.coding/REQUEST.md
.coding/PLAN.md
.coding/VERIFY.md
.coding/SHIP.md
```

Không được còn hành vi lưu vào:

```text
.coding/request.json
.coding/plan.json
.coding/verify.json
.coding/ship.json
```
